01 · ¿Qué es Arduino?

Arduino es una plataforma de electrónica de código abierto basada en hardware y software fáciles de usar. Está diseñada para cualquier persona que quiera crear objetos interactivos o entornos: estudiantes, artistas, aficionados y profesionales. Combina un microcontrolador programable con un entorno de desarrollo sencillo.

⚡

¿Por qué Arduino? Antes de Arduino, crear un dispositivo electrónico interactivo requería conocimientos profundos de electrónica y programación de bajo nivel. Arduino simplificó todo: conectas componentes, escribes unas pocas líneas de código en C/C++ simplificado, y tu proyecto cobra vida en segundos.

Arduino puede leer entradas (un botón, un sensor de temperatura, luz solar) y convertirlas en salidas (activar un motor, encender un LED, publicar datos en internet). Todo esto se controla mediante instrucciones programadas en el Arduino IDE.

02 · Historia de Arduino
2003

Origen en el IDII

Hernando Barragán crea Wiring, el precursor de Arduino, como tesis en el Instituto de Diseño Interactivo de Ivrea, Italia.

2005

Nace Arduino

Massimo Banzi, David Cuartielles y equipo crean Arduino como versión económica y accesible de Wiring para estudiantes.

2008

Arduino Duemilanove

Primera placa con detector automático de USB/alimentación externa. La comunidad crece exponencialmente.

2010

Arduino UNO R1

La placa más icónica de la historia. Se convierte en el estándar mundial de aprendizaje en electrónica.

2015

Arduino MKR / IoT

Se lanza la familia MKR orientada al Internet de las Cosas, con WiFi, Bluetooth y LoRa integrados.

2022

Arduino UNO R4

Nuevo procesador Renesas RA4M1 de 32 bits, mayor velocidad, más memoria y nuevo diseño. La evolución continúa.

03 · Tipos de placas Arduino

Existe una amplia familia de placas Arduino, cada una diseñada para diferentes necesidades. Conocerlas te ayuda a elegir la correcta para tu proyecto.

UNO R3/R4 La más popular

La placa perfecta para comenzar. 14 pines digitales, 6 analógicos, 32 KB de flash. Ideal para proyectos educativos y prototipos.

Microcontrolador: ATmega328P (R3) Voltaje: 5V Velocidad: 16 MHz Flash: 32 KB
MEGA 2560 Para proyectos grandes

54 pines digitales, 16 analógicos y 256 KB de flash. Perfecta para impresoras 3D, CNC y proyectos que necesitan muchos sensores.

Microcontrolador: ATmega2560 Voltaje: 5V Velocidad: 16 MHz Flash: 256 KB
NANO Compacta y versátil

Mismas funciones que el UNO en un tamaño diminuto. Ideal para proyectos donde el espacio importa: wearables, drones, robots pequeños.

Microcontrolador: ATmega328 Voltaje: 5V Tamaño: 18 × 45 mm Conexión: Mini-USB
MKR WiFi 1010 IoT integrado

WiFi y Bluetooth integrados. Procesador de 32 bits ARM Cortex-M0+. Diseñada para proyectos de Internet de las Cosas desde el primer día.

Microcontrolador: SAMD21 Voltaje: 3.3V WiFi: u-blox NINA-W102 Bluetooth: BLE 5.0
LEONARDO USB nativo

Puede emular un teclado o ratón USB directamente. Muy usada para crear controladores de juegos y herramientas de accesibilidad.

Microcontrolador: ATmega32u4 Voltaje: 5V USB nativo: HID (teclado/ratón) Flash: 32 KB
DUE 32 bits, alta velocidad

El primero de 32 bits ARM. Perfecto para procesamiento de audio, visión artificial y aplicaciones que requieren gran potencia de cómputo.

Microcontrolador: AT91SAM3X8E Voltaje: 3.3V Velocidad: 84 MHz Flash: 512 KB
04 · Anatomía del Arduino UNO

El Arduino UNO es la placa de referencia. Conocer cada parte es el primer paso para dominar Arduino.

Microcontrolador
ATmega328P
El cerebro de la placa. Ejecuta el programa cargado. Tiene 32 KB de memoria flash, 2 KB de SRAM y 1 KB de EEPROM.
Pines Digitales
D0 – D13
14 pines que funcionan como entrada o salida digital (0V = apagado / 5V = encendido). Los pines 3,5,6,9,10,11 soportan PWM (~).
Pines Analógicos
A0 – A5
6 pines de entrada analógica. Leen valores de 0 a 5V con una resolución de 10 bits (0-1023). Ideales para sensores de temperatura, luz, etc.
Alimentación
5V, 3.3V, GND, Vin
Pines de alimentación. 5V y 3.3V para alimentar componentes. GND es tierra. Vin permite alimentar la placa externamente (7-12V).
Puerto USB
USB Tipo-B
Sirve para cargar programas desde el PC y para comunicación serie (Serial Monitor). También alimenta la placa a 5V desde el ordenador.
Conector Barrel Jack
Alimentación externa
Acepta entre 7 y 12V CC. Se usa cuando el proyecto necesita más corriente que la que puede dar el USB, como con motores o muchos LEDs.
Cristal de cuarzo
16 MHz
Genera el pulso de reloj que sincroniza todas las operaciones del microcontrolador. Determina la velocidad de ejecución del programa.
LED integrado
Pin 13
LED soldado en la placa conectado al pin 13. Útil para hacer pruebas sin componentes externos. El primer programa "Blink" lo usa.
Botón RESET
Reinicio
Reinicia el microcontrolador y vuelve a ejecutar el programa desde el principio. Útil para depurar o reiniciar el proyecto rápidamente.
05 · Componentes electrónicos básicos

Antes de conectar cualquier componente al Arduino, es importante conocer los elementos pasivos y activos más comunes que forman los circuitos.

LED Diodo emisor de luz

Convierte electricidad en luz. Tiene polaridad: el ánodo (+) va al pin y el cátodo (−) a GND. Siempre usa una resistencia en serie (220–1000 Ω) para no quemarlo.

Voltaje típico: 2V – 3.3V Corriente: 10 – 20 mA Resistencia: (5V - Vled) / 0.02
Resistencia Limitador de corriente

Reduce el flujo de corriente y divide voltaje. Se identifica por sus bandas de colores. Es el componente más usado en electrónica; protege LEDs, establece divisores de voltaje y configura Pull-up/Pull-down.

Ley de Ohm: V = I × R Bandas: Negro=0, Café=1... Tolerancia: Oro=±5%, Plata=±10%
Capacitor Almacena energía

Almacena y libera energía eléctrica. Se usa para filtrar ruido en la alimentación, estabilizar voltajes y crear retardos. Los electrolíticos tienen polaridad.

Unidad: Faradios (F), µF, nF, pF Polaridad: Electrolíticos (+/−) Uso común: Decoupling 100nF
Transistor Interruptor electrónico

Actúa como interruptor o amplificador. Con Arduino se usa para controlar cargas grandes (motores, relés) que consumen más corriente de la que puede dar un pin (máx 40 mA). Tipos: NPN, PNP, MOSFET.

Pines: Base, Colector, Emisor BC547: NPN, 100 mA, 45V 2N2222: NPN, 600 mA, 40V
Pulsador Interruptor momentáneo

Cierra o abre un circuito mientras se presiona. Se conecta con una resistencia Pull-up (10 kΩ) o usando la resistencia interna del Arduino con INPUT_PULLUP.

pinMode(pin, INPUT_PULLUP); if (digitalRead(pin) == LOW) { // botón presionado }
Potenciómetro Resistencia variable

Divide el voltaje de forma ajustable. Al girar el eje, varía la resistencia entre 0 y su valor máximo. Se lee con analogRead() devolviendo valores de 0 a 1023.

int valor = analogRead(A0); // valor: 0 – 1023 (0V – 5V) int brillo = map(valor,0,1023,0,255);
06 · Sensores compatibles

Los sensores son los ojos y oídos de Arduino. Convierten magnitudes físicas del mundo real (temperatura, luz, distancia, movimiento) en señales eléctricas que Arduino puede leer y procesar.

🌡️

Temperatura y humedad — DHT11 / DHT22
El DHT11 mide temperatura (0–50°C) y humedad (20–80%) con precisión básica. El DHT22 es más preciso (−40 a 80°C) y costoso. Usa una sola línea de datos. Requiere la librería DHT sensor library.

Sensor Mide / Detecta Tipo de señal Notas
DHT11 / DHT22 Temperatura y humedad Digital (1-Wire) DHT22 es más preciso; requiere librería
LM35 Temperatura ambiente Analógica 10 mV/°C, simple de usar con analogRead
HC-SR04 Distancia (ultrasonido) Digital (Trigger/Echo) Rango 2 cm – 400 cm; preciso y económico
PIR (HC-SR501) Movimiento infrarrojo Digital (HIGH/LOW) Detecta calor corporal; ajustable sensibilidad
LDR / Fotorresistencia Intensidad de luz Analógica Resistencia varía con la luz; usar divisor de voltaje
MQ-2 / MQ-135 Gases (humo, CO2, butano) Analógica + Digital Requiere calentamiento previo de 1-2 minutos
MPU-6050 Aceleración + Giroscopio I²C 6 ejes; IMU completo en un chip; muy popular
BMP280 / BME280 Presión y altitud I²C / SPI BME280 agrega humedad; alta precisión
Sensor de lluvia Presencia de agua Analógica + Digital Placa con pistas conductoras; detecta conductividad
IR receptor (TSOP38238) Señales de control remoto Digital Requiere librería IRremote; decodifica NEC, Sony, etc.
Micrófono (MAX9814) Sonido / nivel de audio Analógica Con amplificador integrado; conectar a pin analógico
RFID (RC522) Tags NFC / RFID SPI Lee tarjetas de 13.56 MHz; control de acceso
07 · Actuadores y salidas

Los actuadores son los brazos de Arduino: convierten las señales eléctricas en acciones físicas. Desde motores hasta pantallas, permiten que tu proyecto interactúe con el mundo.

Servomotor SG90 Posición angular

Motor que gira entre 0° y 180° con gran precisión. Controlado por señal PWM. Perfecto para brazos robóticos, timones y mecanismos de apertura.

#include <Servo.h> Servo miServo; miServo.attach(9); miServo.write(90); // 90 grados
Motor DC Rotación continua

Gira continuamente. Necesita un controlador (L298N o L293D) ya que Arduino no puede dar suficiente corriente directamente. Se controla velocidad y dirección.

// Con L298N: analogWrite(ENA, 200); // velocidad digitalWrite(IN1, HIGH); digitalWrite(IN2, LOW); // dir
Motor paso a paso Posición exacta

Gira en pasos discretos (ej. 1.8°/paso = 200 pasos por vuelta). Ideal para impresoras 3D, CNC y plotters. Usa driver A4988 o DRV8825.

#include <Stepper.h> Stepper motor(200, 8,9,10,11); motor.setSpeed(60); // RPM motor.step(100);
Relé Interruptor de alta potencia

Permite controlar dispositivos de corriente alterna (220V) desde Arduino (5V). Funciona como un interruptor electromecánico. Usar módulo relé para mayor seguridad.

digitalWrite(RELE, HIGH); // ON delay(1000); digitalWrite(RELE, LOW); // OFF // ¡Cuidado con 220V!
Pantalla LCD 16x2 Visualización de texto

Muestra 2 líneas de 16 caracteres. Puede conectarse en modo de 4 u 8 bits, o con módulo I²C para usar solo 2 cables. Requiere librería LiquidCrystal.

#include <LiquidCrystal_I2C.h> LiquidCrystal_I2C lcd(0x27,16,2); lcd.init(); lcd.backlight(); lcd.print("Hola Arduino!");
Buzzer / Zumbador Salida de sonido

Genera sonidos y melodías mediante señales PWM. El activo suena con solo darle voltaje; el pasivo necesita frecuencia variable para generar tonos.

// Buzzer pasivo con tone(): tone(pin, 440, 500); // La, 500ms delay(600); noTone(pin);
08 · Protocolos de comunicación

Arduino puede comunicarse con otros dispositivos mediante diferentes protocolos. Cada uno tiene ventajas según la distancia, velocidad y número de dispositivos.

ProtocoloDescripciónCablesDispositivos
UART / Serial Comunicación serie asíncrona. Usado para depurar con el Monitor Serie del IDE y para módulos como GPS, Bluetooth HC-05. TX, RX (2) 1 a 1
I²C (TWI) Bus de dos hilos para múltiples dispositivos. Cada uno tiene dirección única. Ideal para sensores: MPU-6050, BMP280, LCD, OLED. SDA, SCL (2) Hasta 127
SPI Bus síncrono de alta velocidad. Más rápido que I²C. Usado para tarjetas SD, pantallas TFT, RFID RC522, DAC. MOSI, MISO, SCK, CS (4) Múltiples
1-Wire Un solo cable de datos + GND. Usado por sensores DHT11/22 y DS18B20 (temperatura). Hasta 100 m de distancia. DATA (1) Múltiples
PWM Modulación por ancho de pulso. Simula salida analógica variando el ciclo de trabajo. Controla brillo de LEDs, velocidad de motores y servos. 1 pin digital 1 actuador/pin
09 · Shields & Módulos populares

Los Shields son placas que se apilan sobre el Arduino añadiendo funcionalidades completas. Los módulos son circuitos listos para conectar con pocos cables.

Módulo Bluetooth HC-05 Comunicación inalámbrica

Añade Bluetooth 2.0 a Arduino. Permite controlar el proyecto desde una app de smartphone. Comunicación por UART (TX/RX). Alcance: ~10 metros.

Serial.begin(9600); char c = Serial.read(); if (c == '1') digitalWrite(LED, HIGH);
Módulo WiFi ESP8266 IoT económico

Agrega conectividad WiFi. Puede funcionar solo (sin Arduino) o como módulo esclavo. El ESP-01 es el más pequeño; el NodeMCU incluye placa de desarrollo.

#include <ESP8266WiFi.h> WiFi.begin("SSID", "pass"); // Conectar al servidor/API
Shield Ethernet W5100 Internet por cable

Conecta Arduino a internet por cable RJ45. Más estable que WiFi. Ideal para servidores web locales, control remoto industrial y datalogging.

#include <Ethernet.h> Ethernet.begin(mac, ip); EthernetServer server(80);
Shield Motor L298N Control de motores

Controla hasta 4 motores DC o 2 motores paso a paso. Soporta hasta 2A por canal. Incluye puente H para inversión de dirección. Esencial en robótica.

// Velocidad: 0–255 (PWM) // Dirección: HIGH/LOW analogWrite(ENA, 150); digitalWrite(IN1, HIGH);
Módulo SD Card Almacenamiento de datos

Lee y escribe en tarjetas microSD. Perfecto para dataloggers que guardan lecturas de sensores con marca de tiempo. Usa protocolo SPI.

#include <SD.h> File archivo = SD.open("datos.txt", FILE_WRITE); archivo.println("temperatura,28");
Pantalla OLED 0.96" Display compacto

Pantalla de 128×64 píxeles, muy brillante y de bajo consumo. Comunicación I²C. Ideal para mostrar datos de sensores, menús o gráficas en proyectos portátiles.

#include <Adafruit_SSD1306.h> display.clearDisplay(); display.println("Temp: 28°C"); display.display();
10 · Guía de aprendizaje paso a paso

Seguir un camino estructurado te permite avanzar de forma sólida. Esta guía lleva a cualquier principiante desde cero hasta proyectos funcionales.

01

Instala el Arduino IDE

Descarga el IDE gratuito desde arduino.cc/software. Disponible para Windows, macOS y Linux. Incluye editor de código, compilador y Serial Monitor. También puedes usar Arduino Web Editor sin instalar nada.

02

Conoce la estructura básica del código

Todo programa Arduino tiene dos funciones obligatorias: setup() que se ejecuta una sola vez al inicio (configuraciones), y loop() que se repite infinitamente (lógica principal). No necesitas main() como en C tradicional.

03

Haz parpadear un LED (Blink)

El "Hola mundo" de Arduino. Usa el LED del pin 13 ya integrado en la placa. Ve a Archivo → Ejemplos → Basics → Blink. Cárgalo y el LED comenzará a parpadear. Modifica los tiempos de delay() para ver los cambios al instante.

04

Aprende a leer entradas digitales y analógicas

Conecta un pulsador y usa digitalRead(). Conecta un potenciómetro y usa analogRead(). Usa el Serial Monitor (Ctrl+Shift+M) para imprimir los valores y entender qué está leyendo Arduino en tiempo real.

05

Controla la intensidad con PWM

Usa analogWrite(pin, 0-255) en los pines marcados con ~ para controlar el brillo de un LED o la velocidad de un motor. Combina con un potenciómetro para controlar en tiempo real.

06

Agrega tu primer sensor

Conecta un DHT11 para leer temperatura y humedad. Instala la librería desde Herramientas → Administrador de Librerías. Las librerías simplifican enormemente el trabajo con sensores complejos; aprende a buscarlas e instalarlas.

07

Construye tu primer proyecto completo

Combina lo aprendido: sensor + actuador + lógica. Ejemplo: termómetro que enciende un ventilador si la temperatura supera un umbral. O estación meteorológica que muestra datos en una pantalla LCD. El aprendizaje se consolida en proyectos reales.

08

Explora comunicación e IoT

Añade módulo WiFi ESP8266 o ESP32 para enviar datos a la nube (Thingspeak, Firebase, MQTT). Controla tu proyecto desde internet o visualiza datos en tiempo real en dashboards como Grafana o plataformas como Blynk.

11 · Código esencial de Arduino

Estas son las funciones y comandos más usados en la programación de Arduino. Dominarlos te da acceso al 80% de los proyectos.

Arduino C++ · Estructura base
// ── Variables globales ────────────────────────────── const int LED_PIN = 13; // Pin del LED integrado const int BTN_PIN = 2; // Pin del pulsador int contador = 0; // Variable de conteo // ── setup(): se ejecuta UNA sola vez ──────────────── void setup() { Serial.begin(9600); // Abre comunicación serie pinMode(LED_PIN, OUTPUT); // Pin 13 como salida pinMode(BTN_PIN, INPUT_PULLUP); // Pin 2 con resistencia pull-up } // ── loop(): se repite INFINITAMENTE ───────────────── void loop() { if (digitalRead(BTN_PIN) == LOW) { // Botón presionado contador++; digitalWrite(LED_PIN, HIGH); // LED encendido Serial.print("Pulsaciones: "); Serial.println(contador); delay(200); // Debounce } else { digitalWrite(LED_PIN, LOW); // LED apagado } }
Arduino C++ · Sensor DHT11 + LCD I²C
#include <DHT.h> #include <LiquidCrystal_I2C.h> DHT dht(4, DHT11); // Sensor en pin 4 LiquidCrystal_I2C lcd(0x27, 16, 2); // LCD I²C dirección 0x27 void setup() { dht.begin(); lcd.init(); lcd.backlight(); } void loop() { float temp = dht.readTemperature(); // Lee temperatura °C float hum = dht.readHumidity(); // Lee humedad % lcd.setCursor(0, 0); lcd.print("Temp: "); lcd.print(temp); lcd.print("C"); lcd.setCursor(0, 1); lcd.print("Hum: "); lcd.print(hum); lcd.print("%"); delay(2000); // Actualiza cada 2 segundos }

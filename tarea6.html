01 · ¿Qué es Python?

Python es un lenguaje de programación de alto nivel, de propósito general, con una sintaxis clara y legible que se asemeja al inglés. Fue diseñado con la filosofía de que el código debe ser fácil de leer y escribir, tanto para principiantes como para expertos.

🐍

¿Por qué se llama Python? Su creador, Guido van Rossum, era fanático del grupo de comedia británico Monty Python's Flying Circus. Eligió ese nombre porque quería algo corto, único y misterioso. La serpiente del logo fue adoptada más tarde como símbolo oficial.

Python es actualmente el lenguaje más popular del mundo según múltiples índices (TIOBE, Stack Overflow). Es el lenguaje número uno en Inteligencia Artificial, Machine Learning, ciencia de datos, automatización y desarrollo web backend.

Interpretado Tipado dinámico Multiparadigma IA & Data Science Automatización Web Backend Open Source Extensión .py
02 · Historia y versiones
1989

Nacimiento de la idea

Guido van Rossum comienza a desarrollar Python durante las vacaciones de Navidad como sucesor del lenguaje ABC.

1991

Python 0.9.0

Primera versión pública publicada. Ya incluía funciones, manejo de excepciones, clases y módulos.

2000

Python 2.0

Introduce list comprehensions, recolección de basura y soporte Unicode. Se vuelve ampliamente adoptado.

2008

Python 3.0

Rediseño importante que rompe compatibilidad con Python 2. Mejoras en Unicode, print como función y más.

2020

Fin de Python 2

Python 2 llega a su fin de vida oficial. Todo el ecosistema migra a Python 3.

2024

Python 3.12 — 3.13

Versiones actuales con mejoras de rendimiento, nuevos mensajes de error y soporte experimental para GIL libre.

04 · Sintaxis básica

La característica más distintiva de Python es que usa indentación (espacios al inicio de la línea) en lugar de llaves { } para definir bloques de código. No usar la indentación correcta causa errores.

⚠️

Regla de oro: En Python el punto y coma ; al final de cada línea no se usa. Los bloques se definen con dos puntos : y la siguiente línea indentada con 4 espacios (o un Tab).

Python — Sintaxis fundamental
# Esto es un comentario de una línea """ Esto es un comentario de múltiples líneas (docstring) """ # print() muestra texto en pantalla print("Hola mundo") print("Hola", "Antonella") # Hola Antonella print(10 + 5) # 15 # input() recibe datos del usuario nombre = input("¿Tu nombre? ") print(f"Hola, {nombre}!") # Indentación: 4 espacios para bloques if True: print("Esto está dentro del bloque") print("Esto está fuera del bloque")
Salida en consola
>>> Hola mundo >>> Hola Antonella >>> 15 >>> ¿Tu nombre? Antonella >>> Hola, Antonella!

08 · Condicionales if / elif / else

Las estructuras condicionales permiten ejecutar código solo cuando se cumple una condición.

09 · Bucles for y while
Python — Bucles for y while
# ── for: itera sobre una secuencia ── for i in range(5): # 0, 1, 2, 3, 4 print(i) for i in range(1, 6): # 1, 2, 3, 4, 5 print(i) # Iterar sobre una lista frutas = ["manzana", "pera", "uva"] for fruta in frutas: print(f"Me gusta la {fruta}") # enumerate: índice + valor for i, fruta in enumerate(frutas): print(f"{i}: {fruta}") # ── while: repite mientras la condición sea True ── contador = 0 while contador < 5: print(f"Vuelta {contador}") contador += 1 # break y continue for n in range(10): if n == 3: continue # salta el 3 if n == 7: break # detiene el bucle en 7 print(n)

Las funciones permiten agrupar código reutilizable. En Python se definen con la palabra clave def.

Python — Funciones

Los diccionarios almacenan pares de clave: valor. Son como un JSON en Python. Son mutables y muy usados para representar objetos.

14 · Buenas prácticas (PEP 8)

PEP 8 es la guía de estilo oficial de Python. Seguirla hace tu código más legible y profesional.

✓ Hacer Nombres descriptivos

Usa snake_case para variables y funciones. Nombres que explican qué hace la variable.

# ✓ Correcto nombre_alumna = "Antonella" def calcular_promedio(notas): return sum(notas) / len(notas) # ✗ Evitar n = "Antonella" def cp(x): return sum(x)/len(x)
✓ Hacer Comentarios útiles

Comenta el por qué, no el qué. El código explica qué hace; el comentario explica por qué.

# ✓ Útil: explica el propósito # Multiplicamos por 2 porque los # precios están en centavos precio_real = precio * 2 # ✗ Inútil: repite lo obvio # Multiplicar precio por 2 precio_real = precio * 2
✓ Hacer Docstrings en funciones

Documenta tus funciones con docstrings para explicar qué hacen, sus parámetros y su retorno.

def saludar(nombre, formal=False): """ Saluda a una persona. Args: nombre: str — nombre formal: bool — si es formal Returns: str — saludo """ if formal: return f"Buenos días, {nombre}" return f"¡Hola, {nombre}!"
✗ Evitar Variables de un solo carácter

Excepto en bucles simples for i in range(), evita nombres de un solo carácter. No describen nada.

# ✗ Inentendible a = 17 b = 9.5 c = a * b # ✓ Claro edad = 17 salario = 9.5 total = edad * salario
✗ Evitar Código duplicado

Si copias y pegas código más de dos veces, créa una función. El principio DRY: Don't Repeat Yourself.

# ✗ Repetido promedio1 = (n1+n2+n3) / 3 promedio2 = (n4+n5+n6) / 3 # ✓ Función reutilizable def promedio(*notas): return sum(notas) / len(notas)
✗ Evitar Ignorar errores silenciosamente

No uses except: sin especificar el tipo de error. Oculta bugs y hace el código difícil de depurar.

# ✗ Peligroso try: resultado = a / b except: pass # ✓ Específico try: resultado = a / b except ZeroDivisionError: print("No se puede dividir entre 0")

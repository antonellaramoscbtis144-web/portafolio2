01 · Introducción

HTML significa HyperText Markup Language (Lenguaje de Marcado de Hipertexto). Es el lenguaje estándar para crear páginas web. No es un lenguaje de programación — no tiene lógica ni condiciones — sino un lenguaje de marcado que define la estructura y el significado del contenido.

💡

Analogía: Si una página web fuera una casa, el HTML sería la estructura (paredes, puertas, ventanas), el CSS sería la decoración (pintura, muebles) y el JavaScript sería la electricidad e instalaciones (lo que hace que las cosas funcionen).

HTML5 es la versión más reciente y estable, lanzada en 2014. Añadió elementos semánticos, soporte nativo para audio y video, APIs de canvas, geolocalización, almacenamiento local y mucho más sin necesitar plugins externos como Flash.

Creado en 1991 por Tim Berners-Lee Estándar W3C Funciona en todos los navegadores Base de toda la web Extensión .html
02 · Estructura base de un documento

Todo documento HTML5 debe comenzar con la misma estructura básica. Esta es la plantilla que usarás en cada página que crees.

HTML — Estructura completa
<!-- Declaración del tipo de documento --> <!DOCTYPE html> <!-- Raíz del documento, define el idioma --> <html lang="es"> <!-- Cabecera: información no visible --> <head> <meta charset="UTF-8"/> <meta name="viewport" content="width=device-width, initial-scale=1.0"/> <title>Mi primera página web</title> <link rel="stylesheet" href="estilos.css"/> </head> <!-- Cuerpo: todo el contenido visible --> <body> <h1>Hola mundo</h1> <p>Este es mi primer párrafo.</p> <script src="script.js"></script> </body> </html>

04 · Etiquetas de texto y tipografía

Las etiquetas de texto definen la jerarquía y el formato del contenido escrito.

Encabezados (headings)
<h1> – <h6> Encabezados

Define títulos de 6 niveles jerárquicos. h1 es el más importante (solo uno por página), h6 el menos relevante.

<h1>Título principal</h1> <h2>Subtítulo</h2> <h3>Sección</h3>
<p> Párrafo

Define un bloque de texto. Es el elemento más usado para contenido. Agrega un salto de línea automático antes y después.

<p>Este es un párrafo.</p> <p>Otro párrafo separado.</p>
<strong> <em> Énfasis

<strong> pone en negrita (importancia). <em> pone en cursiva (énfasis). Ambas tienen valor semántico.

<p> Texto <strong>importante</strong> y <em>énfasis</em>. </p>
<br> <hr> Saltos y líneas

<br> inserta un salto de línea. <hr> dibuja una línea horizontal separadora. Ambas son auto-cerrables.

<p>Línea 1<br>Línea 2</p> <hr> <p>Después del separador</p>
<span> <div> Contenedores

<span> es un contenedor en línea. <div> es un contenedor en bloque. Ninguno tiene significado semántico propio.

<div class="caja"> <p>Texto con <span style="color:red"> rojo </span> </p> </div>
<blockquote> <code> Citas y código

<blockquote> marca citas largas de otra fuente. <code> representa código de computadora en línea.

<blockquote> Frase célebre aquí. </blockquote> <p>Usa <code>console.log()</code></p>

05 · Listas
<ul> Lista desordenada

Crea una lista con viñetas (bullets). Cada elemento va dentro de <li>. El orden no importa.

<ul> <li>HTML</li> <li>CSS</li> <li>JavaScript</li> </ul>
<ol> Lista ordenada

Crea una lista numerada. El orden sí importa: pasos, rankings, instrucciones.

<ol> <li>Primer paso</li> <li>Segundo paso</li> <li>Tercer paso</li> </ol>
<dl> Lista de definición

Para glosarios: <dt> es el término y <dd> es la descripción.

<dl> <dt>HTML</dt> <dd>Lenguaje de marcado</dd> <dt>CSS</dt> <dd>Lenguaje de estilos</dd> </dl>

06 · Enlaces e imágenes
<a> Enlace (anchor)

Crea hipervínculos. href define el destino. target="_blank" abre en nueva pestaña.

<!-- Enlace externo --> <a href="https://google.com" target="_blank"> Ir a Google </a> <!-- Enlace interno a sección --> <a href="#contacto">Contacto</a>
<img> Imagen

Inserta una imagen. Es auto-cerrable. src es la ruta, alt el texto alternativo (obligatorio para accesibilidad).

<img src="foto.jpg" alt="Descripción de la foto" width="300" height="200" />
<figure> Figura con leyenda

Agrupa una imagen con su descripción. <figcaption> añade una leyenda semántica.

<figure> <img src="logo.png" alt="Logo HTML5"/> <figcaption> Logotipo oficial de HTML5 </figcaption> </figure>
07 · Tablas

Las tablas organizan datos en filas y columnas. Nunca se deben usar para maquetar páginas.

HTML — Tabla completa

09 · Etiquetas semánticas HTML5

Las etiquetas semánticas le dicen al navegador y a los buscadores qué significa cada parte de la página, no solo cómo se ve. Mejoran el SEO, la accesibilidad y la legibilidad del código.

"El HTML semántico es la diferencia entre construir una casa con ladrillos sin etiquetar y construir una con planos arquitectónicos."

— Principio de desarrollo web moderno

Estructura semántica de una página

<header> Encabezado del sitio — logo, nav principal
<nav> Navegación principal
<main> Contenido principal (único por página)
<article> Contenido independiente (post, noticia)
<section> Sección temática dentro del artículo
<aside> Contenido lateral (barra, anuncios)
<footer> Pie de página — contacto, derechos
<header>Cabecera

Contiene el encabezado de la página o de una sección: logo, título, menú principal.

<header> <h1>Mi Sitio</h1> <nav>...</nav> </header>
<main>Contenido principal

Engloba el contenido único de la página. Solo puede haber uno por documento.

<main> <h2>Bienvenida</h2> <p>Contenido aquí.</p> </main>
<nav>Navegación

Agrupa los enlaces de navegación principal del sitio o de la página.

<nav> <a href="/">Inicio</a> <a href="/sobre">Sobre mí</a> </nav>
<article>Artículo

Contenido autónomo que tiene sentido por sí solo: una noticia, blog post, comentario.

<article> <h2>Título del post</h2> <p>Contenido...</p> </article>
<section>Sección

Agrupa contenido temáticamente relacionado. Siempre debe tener un encabezado propio.

<section id="contacto"> <h2>Contáctame</h2> <form>...</form> </section>
<footer>Pie de página

Contiene información al final: derechos de autor, contacto, redes sociales.

<footer> <p>© 2025 Antonella</p> </footer>
10 · Multimedia HTML5

HTML5 introdujo soporte nativo para audio y video, eliminando la necesidad de plugins externos como Flash.

<video>Video

Incrusta un video directamente. Acepta múltiples formatos con <source> para compatibilidad.

<video controls width="400" poster="miniatura.jpg"> <source src="video.mp4" type="video/mp4"/> Tu navegador no soporta video. </video>
<audio>Audio

Incrusta audio con controles de reproducción. El atributo controls muestra play/pausa/volumen.

<audio controls loop> <source src="musica.mp3" type="audio/mpeg"/> <source src="musica.ogg" type="audio/ogg"/> </audio>
<iframe>Marco embebido

Incrusta otra página o recurso externo (YouTube, Google Maps, Spotify) dentro de la página.

<!-- Video de YouTube --> <iframe src="https://www.youtube.com/ embed/ID_DEL_VIDEO" width="560" height="315" allowfullscreen ></iframe>
<canvas>Lienzo de dibujo

Área de dibujo controlada por JavaScript. Usada en juegos, gráficas, animaciones y editores de imagen.

<canvas id="miCanvas" width="300" height="150" ></canvas> <!-- Dibujado con JS --> <script> const c = document .getElementById('miCanvas'); </script>

12 · Meta etiquetas esenciales

Las meta etiquetas van dentro del <head> y proveen información sobre la página a navegadores y buscadores.

HTML — Meta etiquetas
<head> <!-- Codificación de caracteres (siempre la primera) --> <meta charset="UTF-8"/> <!-- Responsive: escala correctamente en móviles --> <meta name="viewport" content="width=device-width, initial-scale=1.0"/> <!-- Descripción para buscadores (SEO) --> <meta name="description" content="Portafolio de Antonella Ramos - Cultura Digital 2"/> <!-- Autor de la página --> <meta name="author" content="Fernanda Antonella Ramos Molina"/> <!-- Vista previa en redes sociales (Open Graph) --> <meta property="og:title" content="Mi Portafolio"/> <meta property="og:image" content="preview.jpg"/> <!-- Título que aparece en la pestaña del navegador --> <title>Guía HTML5 · Cultura Digital 2</title> </head>
13 · Buenas prácticas
✓ Hacer Indentación correcta

Usa 2 o 4 espacios para indentar el código. Hace el HTML legible y fácil de mantener.

<ul> <li>Elemento 1</li> <li>Elemento 2</li> </ul>
✓ Hacer Siempre poner alt en imágenes

El atributo alt es obligatorio por accesibilidad. Describe la imagen para lectores de pantalla y cuando no carga.

<!-- Correcto ✓ --> <img src="logo.png" alt="Logo de la escuela"/>
✓ Hacer Un solo H1 por página

Solo debe haber un <h1> por página — el título principal. Los demás títulos siguen la jerarquía h2, h3, etc.

<h1>Título único</h1> <h2>Sección 1</h2> <h3>Subsección</h3> <h2>Sección 2</h2>
✗ Evitar Estilos en línea

Evita usar style="" directamente en el HTML. Los estilos deben ir en un archivo CSS separado para mantener el código limpio.

<!-- Evitar ✗ --> <p style="color:pink;font-size:20px"> Texto </p> <!-- Mejor: usar clase CSS --> <p class="texto-especial">Texto</p>
✗ Evitar Tablas para layout

Las tablas son para datos tabulares, no para maquetar el diseño de la página. Para eso usa CSS Flexbox o Grid.

<!-- MAL: tabla para layout ✗ --> <table><tr><td>menú</td> <td>contenido</td></tr></table> <!-- BIEN: flexbox CSS ✓ --> <div class="flex-layout">...</div>
✗ Evitar Etiquetas de presentación

Etiquetas como <b>, <i>, <font> o <center> son obsoletas. Usa CSS o las semánticas <strong> y <em>.

<!-- Obsoleto ✗ --> <font color="red">Texto</font> <center>Centrado</center> <!-- Correcto ✓ --> <p class="rojo centrado">Texto</p>
